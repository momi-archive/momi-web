# Architecture Decision Records (ADR)

이 폴더는 프로젝트의 주요 기술 결정과 그 근거를 기록합니다.

## ADR이란?

ADR은 아키텍처 결정 기록으로, **왜** 특정 기술적 선택을 했는지 문서화합니다.
나중에 "왜 이렇게 했지?"라는 질문에 답할 수 있게 해줍니다.

## 상태 정의

- **제안됨**: 검토 중인 결정
- **승인됨**: 채택된 결정
- **폐기됨**: 더 이상 유효하지 않은 결정
- **대체됨**: 다른 ADR로 대체된 결정

## 새 ADR 작성

1. [template.md](./template.md)를 복사
2. 파일명: `NNNN-제목.md` (예: `0002-state-management.md`)
3. 내용 작성 후 이 README의 목록에 추가

## 작성 팁

- **컨텍스트**: 결정이 필요했던 배경과 제약조건
- **결정**: 선택한 옵션과 이유
- **결과**: 트레이드오프, 발생한 문제, 해결 방법
