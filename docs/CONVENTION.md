# 프로젝트 컨벤션 (Project Conventions)

## 1. Git 커밋 규칙
- **작업 완료 시**: 의미 있는 작업 단위(기능 구현, 디자인 적용 등)가 끝날 때마다 반드시 커밋 여부를 사용자에게 물어봅니다.
- **언어**: 커밋 메시지는 **한글**로 작성합니다.
- **형식**: `type: subject` 형식을 따르되, 내용은 한글로 적습니다.
  - 예: `feat: 로그인 기능 구현`, `design: 메인 페이지 디자인 시스템 적용`

## 2. 작업 흐름
- 항상 사용자의 승인을 얻은 후 커밋을 진행합니다.

## 3. 코드 품질 체크 (AI 필수 수행)
작업 완료 후 커밋 전에 **반드시** 다음 항목을 확인합니다:

### 3.1 미사용 Import 제거
- 모든 파일에서 사용하지 않는 import 문을 제거합니다.
- TypeScript 빌드 에러(`error TS6192`)를 방지합니다.

### 3.2 타입 체크
- TypeScript 컴파일 에러가 없는지 확인합니다.
- 가능하면 로컬에서 `tsc --noEmit` 또는 `npm run build` 실행을 권장합니다.

### 3.3 Lint 확인
- IDE에서 제공하는 lint 에러를 확인하고 수정합니다.
- 특히 Vercel 배포 시 빌드가 실패하지 않도록 주의합니다.

**중요**: 이 체크는 사용자가 요청하지 않아도 **AI가 자동으로 수행**해야 합니다.
