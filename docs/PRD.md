# 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요 (Project Overview)
- **프로젝트 명**: 나만의 아카이빙 서비스 (가칭: Momi)
- **목적**: 유용한 블로그 글, 이미지, 명령어(코드) 등을 체계적이고 호환성 높게 관리하기 위함.
- **타겟 유저**: 개발자, 디자이너, 학생 등 정보를 수집하고 정리하는 모든 개인 사용자.
- **핵심 가치**: 쉽고 빠른 저장, 강력한 검색, 어디서든 접근 가능(Web/Mobile).

## 2. 사용자 스토리 (User Stories)
1. **빠른 저장**: 사용자는 웹 서핑 중 발견한 URL이나 텍스트를 복사하여 서비스에 빠르게 붙여넣기 할 수 있어야 한다.
2. **콘텐츠 분류**: 사용자는 저장한 콘텐츠를 태그나 폴더를 통해 체계적으로 분류할 수 있어야 한다.
3. **명령어 관리**: 사용자는 자주 쓰는 터미널 명령어나 코드를 저장하고, 버튼 하나로 클립보드에 복사할 수 있어야 한다.
4. **이미지 저장**: 사용자는 스크린샷이나 이미지를 업로드하여 보관할 수 있어야 한다.
5. **검색 및 조회**: 사용자는 키워드 검색을 통해 과거에 저장한 자료를 즉시 찾을 수 있어야 한다.
6. **모바일 접근**: 사용자는 PWA(앱) 환경을 통해 이동 중에도 저장된 내용을 확인하고 관리할 수 있어야 한다.

## 3. 주요 기능 (Key Features)

### 3.1 아이템 저장 (Archive Item)
- **타입 구분**:
  - **링크 (Link)**: URL 입력 시 제목, 설명, 썸네일 등 메타데이터 자동 추출 (Backend 처리).
  - **텍스트 (Text)**: 간단한 메모.
  - **이미지 (Image)**: 파일 업로드 지원.
  - **코드/명령어 (Code/Command)**: 코드 하이라이팅 및 복사 기능.
- **메타데이터**:
  - 제목, 설명, 태그, 생성일/수정일 관리.

### 3.2 대시보드 및 리스트 (Dashboard & List)
- **뷰 모드**: 그리드(카드) / 리스트 뷰 전환.
- **필터링 & 검색**: TanStack Query를 활용한 고성능 필터링 및 실시간 검색.

### 3.3 상세 보기 & 관리
- 아이템 상세 조회 및 CRUD 기능.

### 3.4 모바일 최적화 (Mobile Experience)
- **PWA (Progressive Web App)**: 모바일 브라우저에서 앱처럼 설치 및 구동.
- **반응형 디자인**: 모바일 화면에 최적화된 UI/UX 제공.

## 5. 기술적 고려사항 (Technical Requirements)
- **Frontend**: React (Vite) + TypeScript + Tailwind CSS
- **State/Data**: Zustand + TanStack Query
- **Backend**: Kotlin REST API (별도 서버)
- **Package Manager**: pnpm

## 6. 개발 로드맵 (Roadmap)
1. **1주차**: 프로젝트 셋업 (React/Kotlin), 기술 스택 검증.
2. **2주차**: UI/UX 퍼블리싱 (반응형/PWA 포함) 및 API 설계.
3. **3주차**: 핵심 기능 구현 (CRUD, 이미지 업로드, 필터링).
4. **4주차**: 연동 테스트, 배포 및 최적화.

## 7. 기대 효과
- 카카오톡보다 체계적인 자료 관리.
- PC/Mobile 어디서든 빠른 접근.
- 나만의 지식 저장소 구축.
